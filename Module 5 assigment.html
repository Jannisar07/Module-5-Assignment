Q-2. How to embed audio and video in a webpage?

Answer With the help of audio and vedio tag we can provide any kind of audio and vedio in our webpage.
following are the steps by which we can embed audio and vedio in your webpage:
1. Audio Syntax:
<audio >
< source src="file_name" type="audio_file_type" >
< /audio >
Attributes of <audio > tag:
i. autoplay- When the page is loaded. It specifies to play audio as soon as possible.
ii. controls- It displays audio control.
iii. loop- It will start the audio again when it is finished.
iv. muted- When the page is loaded audio will be automatically muted

2. vedio Syntax:
<vedio >
< source src="file_name" type="video_file_type" >
</vedio >
Attributes of < vedio > tag:
i. autoplay- When the page is loaded. It specifies to play audio as soon as possible.
ii. controls- It displays audio control.
iii. loop- It will start the audio again when it is finished.
iv. muted- When the page is loaded audio will be automatically muted
v. poster- It specifies an image will be shown until video play.

Q-4 Canvas and SVG tags

Answer SVG is an XML-based language for describing 2D graphics.
Canvas generates 2D graphics on the fly (using JavaScript).
Because SVG is built on XML, every element is accessible through the SVG DOM. JavaScript event handlers can be attached to an element.
Each rendered shape is remembered as an object in SVG. If an SVG object's attributes are modified, the browser can automatically re-render the shape.
Canvas is drawn pixel by pixel. The browser forgets about the graphic once it is drawn on canvas. If its position is altered, the entire scene must be recreated, including any items that may have been obscured by the image.
Basics difference between svg and canvas tags
i. Canvas
  • Depending on the resolution
  • There is no support for event handlers.
  • Text rendering capabilities are inadequate.
  • The final image can be saved as.png or.jpg and is ideal for graphic-intensive games.
ii. SVG
  • Resolution independent
  • Support for event handler .
  • Ideal for applications with huge rendering areas (e.g., Google Maps).
  • Not suitable for gaming purposes
